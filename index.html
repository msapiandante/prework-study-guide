HTML

  • HTML is used to create structure of a webpage.
  • HTML stands for hypertext markup language language.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets
  • The a element (which stands for anchor) creates links to the same webpage or other webpages. The href attribute points to the URL for the link.

CSS

  • Cascading Style Sheets (CSS) is a programming language used by developers to define how webpage content is presented to users.
  • CSS defines the colors, aesthetics, visual layout, and organization.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • most common way to style a webpage is by using an external CSS style sheet
  • Declarations contain two important components: the CSS property we want to apply and the value of the property. For example, the declaration color: blue; includes the CSS property color and the value for that property, blue.
  • Just because files are in the same project repository doesn’t mean they know about each other. External CSS style sheet needs to connected to HTML. To do this- use the HTML link element.
  • Assigning colors in CSS can be done in many ways, including by semantic references such as gray or blue. We can also refer to a color by its hexadecimal code, or hex #.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Version control is the practice of tracking changes to our code.
  • git revert: use this command along with first seven characters of SHA to use version control.
  • An SHA, or hash, is actually a long string of numbers or letters,developers often type out only the first seven numbers or letters
  • Git is a version control system that lets you manage and keep track of each version of your project.
  • Github is cloud-based hosting service that allows us to keep track of our projects inside repositories that are stored remotely.
  • Markdown is a lightweight plain-text formatting language. Working in a Markdown file is similar to working with a .doc, except we have to use special syntax for formatting.
  • Github offers a task management tool called GitHub issues, which gives developers and teams an easy way to keep track of key tasks and progress.
  • Developers use repositories, commonly known as repos, as folders to store: 1. Files that contain code. 2. Files that contain other information about the code, like a README file. 3. The version history for all of the files in the repository.
